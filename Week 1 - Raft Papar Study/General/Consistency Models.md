###### 일관성 모델
- 강한 일관성 모델
	- 모든 노드가 항상 최신의 데이터를 보유하도록 보장
	- 데이터 정확성 최우선
	- 네트워크 지연&노드 장애 발생->가용성↓
	- 모든 노드가 동일한 데이터 뷰 유지 -> 추가적인 동기화 작업 필요
- 약한 일관성 모델
	- 노드 간의 데이터 동기화를 즉시 수행X
	- 가용성↑ 일관성 보장X -> 오래된 데이터
- 순차 일관성 모델
- 최종 일관성 모델
	- 모든 노드 일관된 데이터 뷰 보유 -> 보장
	- 가용성 : 강한 일관성 < 최종 일관성
	- 데이터 일관성 : 약한 일관성 < 최종 일관성
